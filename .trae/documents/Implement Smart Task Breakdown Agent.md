基于当前的全栈架构 (React + Hono + oRPC + Drizzle)，我建议开发一个 **“智能任务拆解助手” (Smart Task Breakdown Agent)**。

### **待解决的用户问题**
用户在面对宏大的目标（例如“计划一次日本旅行”或“学习 React”）时，往往因为不知道具体的执行步骤而感到无从下手。手动创建十几个子任务既繁琐又耗时。

### **解决方案**
开发一个 AI 驱动的“智能添加”功能。用户只需输入一个高层目标，AI 将自动：
1.  分析该目标。
2.  将其拆解为 3-5 个具体的、可执行的步骤。
3.  生成待办事项列表，供用户一键添加到任务清单中。

### **实施计划**

#### **第一阶段：基础设施与配置**
1.  **添加依赖**：在 `packages/api` 中安装 `openai` SDK，用于调用大模型能力。
2.  **环境变量配置**：
    *   修改 `packages/infra/alchemy.run.ts`，将 `AI_API_KEY` 暴露给服务端 Worker。
    *   (需要您在 `apps/server/.env` 中填入 OpenAI 或兼容的 API Key)。

#### **第二阶段：后端开发 (API 层)**
1.  **创建 AI 路由**：
    *   新建 `packages/api/src/routers/ai.ts`。
    *   定义 `generatePlan` 接口，接收 `{ goal: string }` 作为输入。
    *   实现调用 LLM 的逻辑，并将返回结果解析为结构化的任务列表。
2.  **注册路由**：
    *   在 `packages/api/src/routers/index.ts` 中将 `ai` 路由注册到主 `appRouter`。

#### **第三阶段：前端开发 (UI/UX)**
1.  **开发“智能添加”组件**：
    *   在待办事项页面添加一个 “✨ 智能添加” 按钮。
    *   创建一个对话框 (Dialog)，允许用户输入他们的目标。
2.  **联调与集成**：
    *   调用 `orpc.ai.generatePlan` 接口获取建议。
    *   展示生成的任务列表（带复选框）。
    *   允许用户点击“添加选中项”，批量调用 `orpc.todo.create` 写入数据库。

这个方案充分利用了当前技术栈的优势（类型安全的 RPC、后端逻辑、AI 集成以及 React UI），能够真正帮助用户降低任务规划的门槛。
